Troubleshooting

Common issues and solutions

Problem: Cannot connect to SSH host

  • Verify hostname is correct (e.g., login.explorer.northeastern.edu)
  • Check that port 22 is accessible from your network
  • Ensure SSH_USERNAME environment variable is set
  • Test connection manually: ssh username@hostname

Problem: Jobs fail to submit to SLURM

  • Check partition name is valid (run sinfo on cluster)
  • Verify time format: D-HH:MM:SS (e.g., 3-00:00:00)
  • Ensure memory format includes 'M' suffix (e.g., 32768M)
  • Check that root path exists on the cluster
  • Verify conda environment name is correct

Problem: Jobs are killed due to insufficient resources

  • Memory: RMG database requires 2-4 GB, total workflow needs 10-32 GB
  • Time: Most models take 1-3 days, allow extra buffer time
  • Monitor job efficiency with seff JOBID after completion
  • Check error logs for "Out of Memory" or "DUE TO TIME LIMIT" messages
